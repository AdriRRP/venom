## Minimum rules

1. Work in `waves` and `slices`.
2. End each `slice` with a verifiable commit.
3. End each `wave` with `scripts/check-wave.sh` and a push.
4. Keep canonical `.feature` files by capability, never by wave.
5. Use `docs/product-direction.md` to discover the next wave when no active wave exists.
6. Update glossary, invariants, scripts, or docs in the same wave when reality changes.

## Collaboration standard

Prefer small, traceable, reviewable changes.
